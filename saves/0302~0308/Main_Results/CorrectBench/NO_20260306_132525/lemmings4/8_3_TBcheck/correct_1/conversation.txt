```python
            print(f"Observed: walk_left={walk_left_observed}, walk_right={walk_right_observed}, aaah={aaah_observed}, digging={digging_observed}")
            return False
        return True
```

### Explanation of Changes:
1. **New Variable `previous_direction_reg`:** This new variable is used to store the Lemming's previous walking direction before it starts falling or digging.
2. **State Transitions:**
   - When the Lemming starts falling or digging, the `previous_direction_reg` is set to the current walking direction.
   - When the Lemming hits the ground again after falling, it resumes the `previous_direction_reg`.
   - When the ground reappears after the Lemming has been digging, it resumes the `previous_direction_reg`.

These changes ensure that the Lemming correctly resumes its previous walking direction after it stops falling or digging, addressing the issues in scenarios 4 and 5.
